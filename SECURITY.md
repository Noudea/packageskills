# Security Policy

## Supported versions

Security fixes are best-effort in the latest code on `main` until the project has stable releases.

## Reporting a vulnerability

Please do not open a public issue for security-sensitive reports.

Instead:

1. Use GitHub private vulnerability reporting if it is available for this repository.
2. If it is not available, contact the maintainers privately through GitHub.

When reporting, include:

- affected package and version
- impact and severity
- reproduction steps or proof of concept
- any suggested remediation

We will acknowledge reports as quickly as possible and coordinate a fix before public disclosure when appropriate.
